About STAR

STAR is an initiative undertaken by DICELab at Royal Holloway University of London, to provide a scalable, decentralised web platform for the creation of Multi-agent Systems. Based on the Ambient Event Calculus , STAR offers developers the complete infrastructure needed to run interactive, large scale simulations and experiments.

STAR favours convention over configuration, and attempts to make it as easy as possible for a developer to get a new agent application up and running. Below are the basic concepts of STAR:

A STAR System is the largest component you deal with when constructing a new application. It acts as a web server, and a host for agent containers. By exposing a comprehensive RESTful API, it is straight forward to interface with STAR from either the browser, or your own external applications.

CONTAINER

A STAR System is composed of a set of containers, which in turn are composed of agents. Each container is self-sufficient, and can be lazily spawned and destroyed depending on the needs of the application. Each container can be equipped with ‘Physics’, determining what kind of actions that agents are allowed to undertake. In large applications, it is not feasible to confine all agents to a single container, the performance of which is determined by the computational power of the underlying system. Instead, it is possible to localise containers (distributed over many machines) into neighbouring sets, which allows them to share their events irrespective of physical location.

AGENT

Agents live in containers, and communicate with one another both synchronously (intra-container) and asynchronously (inter-container, via http) by committing ‘Actions’ aimed at one or more entities. All agents have a URL, which allows both human users and external STAR agents to interact over HTTP. Human users can interact with an agent, for example, by sending a GET request to said URL. This request is then received and responded to like any other action. This can be understood by imagining that every agent has its own ‘web page’ that can be accessed by users in the outside world (in fact, this very webpage is the result of the ‘welcome’ agent in the ‘gui’ container responding to your browser request with valid HTML, which is then rendered by the browser). Agents can be uploaded to a running STAR application and spawned dynamically. In addition, they can be bundled up with Agent-specific resource files which are stored on the filesystem and can be served as part of HTTP responses (the CSS files used to render the styles in this page are an example of this.)

About DICELab

The distributed and intelligent computing environments laboratory (dicelab) at Royal Holloway carries out research that spans theoretical and practical studies of complex applications by combining distributed systems techniques, intelligent agent models, and automated reasoning using computational logic.
